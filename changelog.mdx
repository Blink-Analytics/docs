<Update label="04-02-2026" description="The Organization Update" tags={["Serin v1.2"]}>
  <Frame>
    ![Serin v1.2 Update Image](/images/changelog/Serin_v1.2_Update_Image.png)
  </Frame>

  # New Features

  1) Organization Roles: Added three organization roles: Admin, Manager, and Member, each with different permissions. Learn more at [Roles →](/getting-started/roles)

  2) Application Tracking System
  Both organization representatives and candidates can now track the status of applications.

  3) Apply Feature - Candidates can apply to public jobs if they meet the minimum threshold set by the organization.

  4) Mutable Organizations -Organization representatives can update organization details and invite other users based on their assigned roles. 

  5) Demo Job and Resume - Every account now includes sample jobs and resumes to help users understand how to create jobs and add relevant resumes. 

  6) Payment gateway Integration (Phase I) - Added an initial payment gateway that supports transactions. Subscription plans will be introduced soon.

  7) Job Sharing - Users can share job links through WhatsApp, Gmail, or X.

  8) Resume Based Thresholding - Candidates can apply to a job only if their resume meets the similarity threshold set by the organization. 

  # Feature Updates
  1) Job cards now display the organization logo and key job details in a cleaner, more readable layout.
  2) Organizations can now upload a logo and share more information about themselves on their profile. 
  3) The onboarding flow now asks only for the essential information needed for skill-based job matching. 
  4) Reports now use less raw numbers and focus more on easy-to-understand insights. The layout has also been updated to look more professional.
  5) Public jobs can now be viewed by anyone without needing to sign in.

  # Bugs Fixed
  - The Get Started button on the landing page now correctly takes users to the sign-up page.
  - Updating an organization name during onboarding no longer creates duplicate organizations.
  - Phone numbers are now properly validated on the profile page.
  - Users can crop their profile picture before saving it.
  - Improved visibility of the job invitation button in email invites.
  - The organization selector dropdown now closes when clicking outside of it.
  - GitHub and LinkedIn profile badges are now positioned correctly.
  - Removed the job and resume selection step during interview setup.
  - Removed the Interviews tab from organization and candidate views.
  - Users are now redirected to the waitlist page after successful signup.
  - Organization members can no longer create new organizations.
  - The primary (first) organization can no longer be deleted.
  - Landing page scroll indicator now navigates to all sections correctly.
  - No navigation tab is highlighted while viewing the profile page.
  - Newly uploaded resumes now appear instantly when applying for a job.
  - Organization representatives can now test a job from the job details page.
  - Removed the option to activate or deactivate jobs.
  - Average scores shown on Analysis page cards now display the correct value.
  
</Update>


<Update label="04-01-2026" description="The Resolution Update" tags={["Serin v1.1"]}>
  <Frame>
    ![Serin v1.1 Update Image](/images/changelog/Serin_v1.1_Update_Image.png)
  </Frame>

  # New Features
  1) "Objectives" - Every job now includes a set of objectives that define what success looks like for that role. These objectives help generate evaluation rubrics and improve how candidate skills are matched against job requirements. 
  
  2) Public and Private Jobs - You can now control who can see your jobs.
        - Public jobs are visible to anyone on the internet.
        - Private jobs are only visible to you and candidates you explicitly invite.

  3) Github Integration - Users who link their GitHub account can display their repositories, contribution heatmap, and primary programming languages directly on their profile. 

  4) Forgot Password - Users can now reset their password if they forget it.

  5) Interview Validity (Job Expiry) - Organizations can set an expiry period for jobs. Once the expiry time is reached, interview sessions for that job are automatically marked as expired.

  # Feature Updates
  1) Report Update - Reports now evaluate candidate–job fit based on job objectives and the candidate’s responses to objective-driven interview questions.

  2) Landing Page UI Improvements - The landing page has been simplified and optimized for a smoother browsing experience.

  # Bugs Fixed
  - Job page KPIs now accurately reflect real data.
  - Jobs can be edited based on the organization selected in the organization switcher.
  - Fixed issues where updates did not reflect correctly when using multiple organizations.
  - Removed the Help button from the interview room.
  - Job details now ignore empty skills, objectives, and benefits fields.
  - “Invite Candidates” is now limited to jobs belonging to the user’s active organization.
  - Fixed the Applications section in Candidate view so jobs open correctly and display the right job and company names.

</Update>
